# Floor Object Acceptance Gate

## Intent

Floor objects must support the illusion that the player is walking through operational spaces digitally reconstructed from memory.

Objects should imply prior human activity.

## Current Verdict

Status: **Accepted after FPS000-VISUAL-003D**, with future stacking rule pending.

## Accepted Evidence

Player reported:

1. Cabinet vs crate instantly recognizable.
2. Objects feel grounded.
3. Objects block fairly.
4. Drones respect objects.
5. Clusters help the room feel real.
6. No object annoyingly blocks shooting/movement.

## Required Rules

- Objects are solid.
- Drones collide with objects.
- Objects are non-movable.
- Objects are reachability-safe.
- Objects must not block spawn, doors, Gate, Automaton, Stations, or pickups.
- Objects should be clustered into operational zones.
- Objects must feel grounded on the floor plane.

## Stacking Rule

Only like-item stacking is allowed.

Allowed:

- crate stack with crate
- pallet stack with pallet/crate
- rack row with rack
- cabinet bank with cabinet
- server/data block row with server/data block

Forbidden:

- random mixed stacks
- cabinet on crate
- console stacked with rack
- stacked objects near Gate, Stations, pickups, doors, spawn, or required objectives
